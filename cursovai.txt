# Подробное описание функционала и техническая реализация проекта "MedKit"

Данный проект представляет собой веб-приложение на базе FastAPI для автоматизации работы медицинских учреждений в части выписки и контроля электронных рецептов.

---

## 1. Архитектура базы данных (ORM Модели)
Система построена на использовании SQLAlchemy, что позволяет описывать структуру таблиц в виде классов Python.

**Код моделей (`src/models/users.py`):**
```python
class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True, index=True)
    username = Column(String, index=True)
    mail = Column(String, unique=True, index=True)
    hashed_password = Column(String)
    role = Column(String, default="patient")

class Prescription(Base):
    __tablename__ = "prescriptions"
    id = Column(Integer, primary_key=True, index=True)
    doctor_id = Column(Integer, ForeignKey("users.id"))
    patient_id = Column(Integer, ForeignKey("users.id"))
    medication_id = Column(Integer, ForeignKey("medications.id"))
    instruction = Column(String)
    is_issued = Column(Boolean, default=False)
    end_date = Column(DateTime(timezone=True))
```

**Разбор кода:**
*   **Связи (Foreign Keys):** Таблица `Prescription` связывает врачей, пациентов и медикаменты. Это классическая связь "многие-ко-многим", реализованная через промежуточную таблицу назначений.
*   **Безопасность:** Пароли хранятся в виде хешей (`hashed_password`), а не в открытом виде.
*   **Роли:** Поле `role` определяет уровень доступа пользователя в системе.

## 2. Система аутентификации и безопасности (JWT)
Для защиты данных используется стандарт JWT (JSON Web Token). Пользователь получает токен при входе и отправляет его в заголовке каждого запроса.

**Функция проверки текущего пользователя (`src/dependencies.py`):**
```python
def get_current_user(token: Annotated[str, Depends(oauth2_scheme)], db: db_dep):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username: str = payload.get("sub")
        if username is None:
            raise credentials_exception
    except jwt.PyJWTError:
        raise credentials_exception

    user = db.query(users.User).filter(users.User.username == username).first()
    return user
```

**Разбор кода:**
*   **`jwt.decode`:** Декодирует токен и проверяет его подлинность с помощью секретного ключа сервера.
*   **Dependency Injection:** Функция используется как зависимость в роутах, гарантируя, что доступ получат только авторизованные пользователи.

## 3. Разграничение прав доступа (Role-Based Access Control)
Для реализации разных уровней доступа (Врач, Админ) используется специальный класс-чекер.

**Код проверки ролей (`src/dependencies.py`):**
```python
class RoleChecker:
    def __init__(self, need_role: str):
        self.need_role = need_role

    def __call__(self, current_user: Annotated[users.User, Depends(get_current_user)]):
        if current_user.role != self.need_role:
            raise HTTPException(
                status_code=status.HTTP_403_FORBIDDEN, 
                detail="У вас недостаточно прав"
            )
        return current_user
```

**Пример использования в API (`src/api/staff.py`):**
```python
@router_staff.get("/home/admin_panel")
def admin_panel(current_role: Annotated[users.User, Depends(RoleChecker(need_role="Admin"))]):
    return {"role": current_role.role}
```

**Разбор кода:**
*   Этот механизм позволяет одной строчкой кода закрыть доступ к любому эндпоинту для всех, кроме обладателей конкретной роли.

## 4. Бизнес-логика: Создание рецептов
Врачи могут выписывать рецепты пациентам, указывая инструкции и срок действия.

**Код создания рецепта (`src/api/staff.py`):**
```python
@router_staff.post("/home/new_drug_prescription")
def add_drug_prescription(
    DataDrugPrescription: DrugPrescriptionCreate,
    db: Session= Depends(get_db),
    current_user: users.User = Depends(RoleChecker(need_role="Doctor"))
):
    new_drug_prescription = users.Prescription(
        patient_id=DataDrugPrescription.patient_id,
        doctor_id=current_user.id, # ID врача берется из токена
        pharmacy_id=DataDrugPrescription.pharmacy_id,
        medication_id=DataDrugPrescription.medication_id,
        instruction=DataDrugPrescription.instruction,
        quantity=DataDrugPrescription.quantity,
        end_date=DataDrugPrescription.end_date
    )
    db.add(new_drug_prescription)
    db.commit()
```

**Разбор кода:**
*   **Валидация:** Используется схема `DrugPrescriptionCreate` (Pydantic) для проверки входящих данных.
*   **Автоматизация:** `doctor_id` заполняется автоматически из данных текущего авторизованного врача, что исключает подделку авторства рецепта.

---

## 5. Получение данных пациентом
Пациент может видеть только свои назначения, что обеспечивается фильтрацией по его ID.

**Код получения списка рецептов (`src/api/user.py`):**
```python
@router.get("/home/my_prescriptions")
def get_my_active_prescriptions(
    db: Session = Depends(get_db),
    current_user: users.User = Depends(get_current_user)
):
    prescriptions = (
        db.query(...)
        .outerjoin(users.User, users.Prescription.patient_id == users.User.id)
        .filter(users.Prescription.patient_id == current_user.id)
        .all()
    )
    return prescriptions
```

**Разбор кода:**
*   **JOIN:** Запрос объединяет данные из нескольких таблиц (Рецепты, Аптеки, Лекарства), чтобы пользователь видел понятные названия вместо ID.
*   **Безопасность:** Фильтр `.filter(patient_id == current_user.id)` гарантирует, что один пациент не сможет подсмотреть рецепты другого.
